"Switch" ifodasi

Kod yozishda shartli ifodalardan foydalanish davomida ba'zan quyidagiga o‘xshash kod bilan to‘qnashishingiz mumkin.

var option = 3;

if (option === 1) {
console.log("You selected option 1.");
} else if (option === 2) {
console.log("You selected option 2.");
} else if (option === 3) {
console.log("You selected option 3.");
} else if (option === 4) {
console.log("You selected option 4.");
} else if (option === 5) {
console.log("You selected option 5.");
} else if (option === 6) {
console.log("You selected option 6.");
}

Asosan, ushbu har bir shartli ifoda bitta o‘zgaruvchida ma'lum bir qiymatni tekshiradi va agar u rost bo‘lsa, `if` bayonoti ichidagi kod bajariladi. Bu qanday ishlashini ko‘rsatish uchun keling, ushbu kodni ishga tushiramiz va uning natijasini ko‘rib chiqamiz.


Demak, o‘zgaruvchi `option` 3 ga teng bo‘lgani sababli, u "Siz 3-variantni tanladingiz"ni chop etdi. Bu unchalik murakkab emas. Endi, sizga shartli ifodadan foydalanmasdan shu natijaga erishishning boshqa usuli mavjudligini aytsam-chi? Aslida, shunday boshqa usul mavjud, ya'ni `switch` bayonotidan foydalanish. `Switch` bayonoti faqat shu kabi vaziyatlarda, ya'ni bitta o‘zgaruvchi qiymatiga asoslangan ulashgan `if` bayonotlari bo‘lganda ishlaydi. Keling, ushbu kodni `switch` bayonoti yordamida qayta yozib chiqaman va uning qanday ishlashini tushuntiraman. Shuningdek, shartli ifoda versiyasini boshqa tomonda qoldiraman, shunda farqni ko‘rishingiz mumkin.

var option = 3;

switch (option) {
case 1:
console.log("You selected option 1.");
case 2:
console.log("You selected option 2.");
case 3:
console.log("You selected option 3.");
case 4:
console.log("You selected option 4.");
case 5:
console.log("You selected option 5.");
case 6:
console.log("You selected option 6.");
}

Yaxshi, men barcha kodimni `switch` bayonoti ichiga o‘rab chiqdim. Va xuddi o‘qiganingizdek, men kodning qaysi qismi `option` o‘zgaruvchisiga asoslanib bajarilishini o‘zgartirmoqchiman. Shuningdek, har bir `if` bayonotini `case` bandiga almashtirdim, bu yerda `case` qiymati o‘zgaruvchi `option`ning o‘sha qiymatiga mos kelishi kerak, va uning ostida joylashgan kod bajariladi. Asosan, agar `option` qiymati, aytaylik, 3 bo‘lsa va kodimiz `switch` bayonotiga yetib borgan bo‘lsa, u shu `case` mavjudligini tekshiradi. Agar mavjud bo‘lsa, bu holatda bor, u nazoratni o‘sha `case`ga o‘tkazadi va bajarishni davom ettiradi. Agar mavjud bo‘lmasa, u `switch` bayonotini butunlay o‘tkazib yuboradi. Bu qanday ishlashini ko‘rsatish uchun, keling, ushbu kodni ishga tushiramiz va uning bizga qanday natija berishini ko‘raylik.


Mana bu qiziq! "Siz 3-variantni tanladingiz"ni chiqarish bilan birga 4-, 5- va 6-variantlarni ham chop etdi. Nima uchun bu sodir bo‘ldi? Buni tushunish uchun, `switch` bayonotidan foydalanganda nima sodir bo‘layotganini aniq bilish muhimdir.
Eslab qoling, kodimiz `switch` bayonotiga yetib borganida, u o‘zgartirilayotgan o‘zgaruvchi uchun `case` mavjudligini tekshiradi. Agar mavjud bo‘lsa, u nazoratni o‘sha `case`ga o‘tkazadi va bajarishni davom ettiradi. Ammo bu quyidagi `case`larning bajarilmasligini oldini olmaydi. Bu xulq-atvor "qisqa tutashuv" (falling through) deb ataladi. Siz `switch` bayonotini o‘zgaruvchi qiymatiga asoslanib kod qatoriga "sakrash"ga o‘xshash deb o‘ylashingiz mumkin. Shu holatda, `option` 3 ga teng bo‘lgani uchun, biz 3-ga mos keladigan `case`ga "sakraymiz", keyin kodimizni ishga tushirishni davom ettiramiz. Shuning uchun boshqa chop etish bayonotlari ham bajariladi. Agar siz haqiqatan ham `case` bajarilgandan keyin to‘xtashni xohlasangiz, unda `break` kalit so‘zidan foydalanishingiz kerak bo‘ladi.

var option = 3;

switch (option) {
case 1:
console.log("Siz 1 variantni tanlaysiz.");
break;
case 2:
console.log("Siz 2 variantni tanlaysiz.");
break;
case 3:
console.log("You selected option 3.");
break;
case 4:
console.log("You selected option 4.");
break;
case 5:
console.log("You selected option 5.");
break;
case 6:
console.log("You selected option 6.");
break; // technically, not needed
}

`Break` bayonotidan `switch` bayonotidan chiqish uchun foydalanish mumkin, shunda boshqa hech qanday `case`lar bajarilmaydi. Agar men har bir `case`ga `break` bayonotlarini qo‘shsam va keyin ushbu kodni qaytadan ishga tushirsam, faqat "Siz 3-variantni tanladingiz"ni olamiz. Shuningdek, texnik jihatdan, `switch` bayonotining oxirgi holatida (`case`) `break` bayonoti shart emas, chunki bu `switch` ichidagi oxirgi `case`. Shunday qilib, bundan quyidagilarni olib chiqish mumkin: `Switch` bayonotlari faqat o‘zgaruvchi qiymatiga asoslangan kod bayonotlarini bajarishingiz kerak bo‘lgan vaziyatlarda ishlaydi. Bu shunchaki yana bir muqobil usul va uni vaqti-vaqti bilan ko‘rishingiz mumkin. Shuningdek, `switch` bayonotidan faqat raqamlar bilan emas, har qanday ma'lumot turida foydalanishingiz mumkin. Bundan tashqari, `switch` bayonotining qisqa tutashuv (falling through) xulq-atvoridan foydalanishingiz kerak bo‘lgan ba'zi holatlar ham mavjud. Biz buni keyingi bo‘limda batafsil ko‘rib chiqamiz.

Agar kodingizda har bir shart bir xil qiymatga asoslangan bo‘lsa, "else if" ifodalarini takrorlayotgan bo‘lsangiz, u holda "switch" ifodasidan foydalanish vaqti keldi.

if (option === 1) {
console.log("You selected option 1.");
} else if (option === 2) {
console.log("You selected option 2.");
} else if (option === 3) {
console.log("You selected option 3.");
} else if (option === 4) {
console.log("You selected option 4.");
} else if (option === 5) {
console.log("You selected option 5.");
} else if (option === 6) {
console.log("You selected option 6.");
}

"Switch" ifodasi

Agar switch ifodasi shartli ifodalarni ishlatmasdan bir xil qiymatga asoslangan, bir nechta else if ifodalarini bog‘lashning yana bir usuli. Buning o‘rniga, siz shunchaki "switch" qiymatiga qarab qaysi kod bajarilishini bilib olasiz.

switch (option) {
case 1:
console.log("You selected option 1.");
case 2:
console.log("You selected option 2.");
case 3:
console.log("You selected option 3.");
case 4:
console.log("You selected option 4.");
case 5:
console.log("You selected option 5.");
case 6:
console.log("You selected option 6.");
}

Bu yerda har bir else if ifodasi (option === [value]) case jumlasi (case: [value]) bilan almashtirildi va bu jumlalar "switch" ifodasi ichiga joylashtirilgan.

"Switch" ifodasi birinchi baholanganda, u birinchi case bandini qidiradi, uning ifodasi "switch" ifodasiga berilgan ifoda natijasi bilan bir xil qiymatga baholanadi. So‘ngra tegishli boshqaruvlarni bajarib, boshqaruvni ushbu case bandiga o‘tkazadi.

Shunday qilib, agar siz optionni 3ga tenglashtirsangiz...

var option = 3;

switch (option) {
...
}

Prints:
You selected option 3.
You selected option 4.
You selected option 5.
You selected option 6.

..."switch" ifodasi 3, 4, 5 va 6-variantlarni chiqaradi.

Ammo bu yuqoridagi kodning asl if ... else kodi bilan bir xil emasmi? Xo‘sh, nima yetishmayapti?

"Break" ifodasi

var option = 3;

switch (option) {
case 1:
console.log("You selected option 1.");
break;
case 2:
console.log("You selected option 2.");
break;
case 3:
console.log("Siz 3 variantini tanlaysiz.");
break;
case 4:
console.log("You selected option 4.");
break;
case 5:
console.log("You selected option 5.");
break;
case 6:
console.log("You selected option 6.");
