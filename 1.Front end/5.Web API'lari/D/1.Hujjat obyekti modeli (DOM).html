JavaScript - sodda

  • Hujjat obyekti modeli (DOM)

Document Object Model ( DOM ) veb-sahifalarni skriptlar yoki dasturlash tillariga xotirada hujjatning tuzilishini - masalan, veb-sahifani ifodalovchi HTMLni - ifodalash orqali bog'laydi. Odatda bu JavaScriptga ishora qiladi, garchi HTML, SVG yoki XML hujjatlarini obyekt sifatida modellashtirish asosiy JavaScript tilining bir qismi bo'lmasa ham.

DOM mantiqiy daraxtga ega hujjatni ifodalaydi. Daraxtning har bir shoxi tugun bilan tugaydi va har bir tugun obyektlarni o'z ichiga oladi. DOM usullari daraxtga dasturiy kirish imkonini beradi. Ular yordamida siz hujjatning tuzilishini, uslubini yoki tarkibini o'zgartirishingiz mumkin.

Tugunlarga shuningdek, hodisalarni qayta ishlovchi qurilmalar biriktirilgan bo'lishi mumkin. Hodisa ishga tushirilgandan so'ng, hodisalarni qayta ishlovchi qurilmalar bajariladi.

Kontseptsiyalar va qo'llanilishi

Document Object Model (DOM) veb-hujjatlar uchun dasturlash interfeysi hisoblanadi. U sahifani ifodalaydi, shunda dasturlar hujjat tuzilishini, uslubini va mazmunini o'zgartirishi mumkin. DOM hujjatni tugunlar va obyektlar sifatida ifodalaydi; shu tariqa dasturlash tillari sahifa bilan o'zaro ta'sir qilishi mumkin.

Veb-sahifa - bu brauzer oynasida yoki HTML manbai sifatida ko'rsatilishi mumkin bo'lgan hujjat. Ikkala holatda ham u bir xil hujjat, ammo Document Object Model (DOM) tasviri uni boshqarishga imkon beradi. Veb-sahifaning obyektga yo'naltirilgan tasviri sifatida uni JavaScript kabi skript tili bilan o'zgartirish mumkin.

Masalan, DOM ushbu kod parchasidagi usul hujjatdagi querySelectorAll barcha elementlarning ro'yxatini qaytarishi kerakligini belgilaydi:<p>

JS

                            const paragraphs = document.querySelectorAll("p");
                            // paragraphs[0] is the first <p> element
                            // paragraphs[1] is the second <p> element, etc.
                            alert(paragraphs[0].nodeName);

Veb-sahifalarni boshqarish va yaratish uchun mavjud bo'lgan barcha xususiyatlar, usullar va hodisalar obyektlarga ajratilgan. Masalan, document hujjatning o'zini ifodalovchi obyekt, HTML jadvallariga kirish uchun DOM interfeysini table amalga oshiradigan har qanday obyektlar HTML Table Element va boshqalar barchasi obyektlardir.

DOM birgalikda ishlaydigan bir nechta APIlar yordamida qurilgan. Asosiy DOM har qanday hujjatni va uning ichidagi obyektlarni tavsiflovchi obyektlarni belgilaydi. Bu DOMga yangi xususiyatlar va imkoniyatlarni qo'shadigan boshqa APIlar tomonidan kerak bo'lganda kengaytiriladi. Masalan, HTML DOM API asosiy DOMga HTML hujjatlarini ifodalash uchun qo'llab-quvvatlaydi va SVG API SVG hujjatlarini ifodalash uchun qo'llab-quvvatlaydi.

DOM daraxti nima?

DOM daraxti - bu tugunlari HTML yoki XML hujjatining tarkibini ifodalovchi daraxt tuzilishi . Har bir HTML yoki XML hujjatida DOM daraxti tasviri mavjud. Masalan, quyidagi hujjatni ko'rib chiqing:

HTML 

                            <html lang="en">
                               <head>
                                  <title>My Document</title>
                               </head>
                               <body>
                                  <h1>Header</h1>
                                  <p>Paragraph</p>
                               </body>
                            </html>

Yuqoridagi daraxt yuqoridagi hujjatning DOM daraxtiga o'xshash bo'lsa-da, ular bir xil emas, chunki haqiqiy DOM daraxti bo'sh joyni saqlab qoladi .

Veb-brauzer HTML hujjatini tahlil qilganda, u DOM daraxtini yaratadi va keyin uni hujjatni ko'rsatish uchun ishlatadi.

DOM va JavaScript

Oldingi qisqa misol, deyarli barcha misollar singari, JavaScript. Ya'ni, u JavaScriptda yozilgan , lekin hujjat va uning elementlariga kirish uchun DOMdan foydalanadi . DOM dasturlash tili emas, lekin usiz JavaScript tilida veb-sahifalar, HTML hujjatlari, SVG hujjatlari va ularning tarkibiy qismlari haqida hech qanday model yoki tushuncha bo'lmas edi. Hujjatning umumiy ko'rinishi, sarlavha, hujjat ichidagi jadvallar, jadval sarlavhalari, jadval katakchalari ichidagi matn va hujjatdagi boshqa barcha elementlar ushbu hujjat uchun hujjat obyekti modelining qismlari hisoblanadi. Ularning barchasiga DOM va JavaScript kabi skript tili yordamida kirish va ularni boshqarish mumkin.

DOM JavaScript tilining bir qismi emas, balki veb-saytlar yaratish uchun ishlatiladigan veb-API hisoblanadi. JavaScript boshqa kontekstlarda ham ishlatilishi mumkin. Masalan, Node.js kompyuterda JavaScript dasturlarini ishga tushiradi, lekin boshqa API to'plamini taqdim etadi va DOM API Node.js ish vaqtining asosiy qismi emas.

DOM biron bir dasturlash tilidan mustaqil bo'lish uchun yaratilgan bo'lib, hujjatning strukturaviy ko'rinishini bitta, izchil API orqali taqdim etish mumkin edi. Ko'pgina veb-ishlab chiquvchilar DOMdan faqat JavaScript orqali foydalansalar ham, DOM implementatsiyalari har qanday til uchun yaratilishi mumkin, bu Python misolida ko'rsatilgan:

Python

                            # Python DOM misoli 
                            import xml.dom.minidom as m
                            doc = m.parse(r"C:\Projects\Py\chap1.xml")
                            doc.nodeName # DOM property of document object
                            p_list = doc.getElementsByTagName("para")

Vebda JavaScript yozishda qanday texnologiyalar ishtirok etishi haqida ko'proq ma'lumot olish uchun JavaScript texnologiyalariga umumiy nuqtai nazarga qarang.

DOMga kirish

DOM dan foydalanishni boshlash uchun siz biron bir maxsus ish qilishingiz shart emas. Siz API dan to'g'ridan-to'g'ri JavaScript da, brauzer tomonidan boshqariladigan dastur ichidan foydalanasiz.

Script yaratganingizda, u <script> element ichida yoki veb-sahifaga kiritilgan bo'lsin, darhol hujjatning o'zini yoki veb-sahifadagi turli elementlarni (hujjatning avlod elementlari) boshqarish uchun documentyoki windowobyektlari uchun API dan foydalanishni boshlashingiz mumkin. Sizning DOM dasturlashingiz quyidagi misol kabi oddiy bo'lishi mumkin, u funksiyadan foydalanib konsolda xabarni ko'rsatadi console.log():

HTML

                            <body onload="console.log('Welcome to my home page!');">
                              …
                            </body>

Sahifa tuzilishini (HTMLda yozilgan) va DOM manipulyatsiyasini (JavaScriptda yozilgan) aralashtirish odatda tavsiya etilmasligi sababli, JavaScript qismlari bu yerda birlashtiriladi va HTMLdan ajratiladi.

Masalan, quyidagi funksiya yangi h1 elementini yaratadi, shu elementga matn qo'shadi va keyin uni hujjat daraxtiga qo'shadi:

HTML

                            <html lang="en">
                              <head> </head>
                              <body>
                                <script>
                                  // create a couple of elements in an otherwise empty HTML page
                                  const heading = document.createElement("h1");
                                  const headingText = document.createTextNode("Big Head!");
                                  heading.appendChild(headingText);
                                  document.body.appendChild(heading);
                                </script>
                              </body>
                            </html>

  • CDATASection

  • CustomEvent

Ushbu qo'llanma DOM ierarxiyasini boshqarish uchun foydalanishingiz mumkin bo'lgan obyektlar va haqiqiy narsalar haqida. Ularning qanday ishlashini tushunish chalkash bo'lishi mumkin bo'lgan ko'plab jihatlar mavjud. Masalan, HTML elementini ifodalovchi obyekt o'z name xususiyatini HTMLFormElement interfeysdan, className xususiyatini esa HTMLElement interfeysdan form oladi. Ikkala holatda ham, kerakli xususiyat o'sha form obyektida bo'ladi.
